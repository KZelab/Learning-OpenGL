Lab 05 - Index Buffers and Uniforms

Overview

Lab 04 exposed two problems. The vertex buffer wasted memory repeating two corner positions, and the quad's colour was fixed at compile time inside the GLSL source. This lab solves both. An Index Buffer Object (IBO) lets the GPU reuse vertex positions by number rather than repeating them, shrinking the quad from 6 stored vertices to 4. A uniform variable opens a data channel from C++ into the running shader, so the colour can be chosen and changed at runtime without touching the GLSL file.

This lab follows directly from the lecture on shaders and uniforms. The lecture covered the theory. This lab is the practical implementation.

Prerequisites

  • Lab 04 complete: a quad renders as two triangles with 6 vertices and glDrawArrays.
  • src/main.cpp contains parseShader, compileShader, and createProgram from Lab 03.
  • res/shaders/triangle.shader exists and loads correctly.

Learning Objectives

By the end of this lab you will be able to:

  • Create an Index Buffer Object and explain how it eliminates duplicated vertex data.
  • Use glDrawElements in place of glDrawArrays and explain the difference between vertex count and index count.
  • Declare a uniform variable in GLSL and explain how it differs from a vertex attribute.
  • Retrieve a uniform location with glGetUniformLocation and write a value with glUniform4f.
  • Diagnose the two most common uniform mistakes: querying the location before the program is linked, and calling glUniform without an active program.

Concept

Lab 04 stored 6 vertices to draw a quad, repeating two corner positions to satisfy glDrawArrays. An Index Buffer Object (IBO) removes the duplication. Instead of listing every vertex in draw order, you store only the 4 unique positions and supply a separate array of integer indices. Each index is the 0-based slot of a position in the VBO. The GPU assembles triangles by looking up positions by index rather than reading them sequentially. A quad needs 6 indices but only 4 stored positions: the saving grows substantially for dense meshes where every vertex is shared by an average of six triangles.

An IBO is a standard buffer object bound to GL_ELEMENT_ARRAY_BUFFER instead of GL_ARRAY_BUFFER. The VAO records the element buffer binding as part of its state, so rebinding the VAO in the render loop restores it automatically. The draw call changes from glDrawArrays to glDrawElements(GL_TRIANGLES, 6, GL_UNSIGNED_INT, 0): the second argument is the number of indices, not vertices.

A uniform is a value passed from C++ to a shader that stays constant for every vertex and fragment processed in a single draw call. Unlike a vertex attribute, which changes per vertex, a uniform is global to the program. You declare it in GLSL with uniform type name, then from C++ retrieve its location once with glGetUniformLocation and write a value each frame with the matching glUniform* variant. The location must be retrieved after the program is linked. The value must be set while the program is active, so call glUseProgram first. A return value of -1 from glGetUniformLocation means the name was not found - check for typos or confirm the uniform is actually used in the shader.

Why before the out declaration? Uniform declarations are global to the shader and conventionally placed at the top of the source, before any out or function definitions. The GLSL compiler does not enforce the order, but keeping uniforms at the top makes them easy to find.

Task 3 - Define the Index Array

Add the index array immediately after the vertex array. Each row defines one triangle by listing three vertex slot numbers. The positions themselves are not repeated - only the slot references are. 

// file: src/main.cpp -- add after vertices[]
unsigned int indices[] = {
    0, 1, 2,   // triangle 1: bottom-left -> bottom-right -> top-right
    2, 3, 0    // triangle 2: top-right -> top-left -> bottom-left
};

Task 4 - Create and Upload the IBO

Add the IBO setup after the glVertexAttribPointer and glEnableVertexAttribArray calls, while the VAO is still bound. The VAO stores the element buffer binding as part of its state.

  1. Declare unsigned int ibo.
  2. Call glGenBuffers(1, &ibo).
  3. Call glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, ibo).
  4. Call glBufferData(GL_ELEMENT_ARRAY_BUFFER, sizeof(indices), indices, GL_STATIC_DRAW).
  5. Now unbind the VAO with glBindVertexArray(0).

glBindVertexArray(0);  // unbind VAO after the IBO is bound, not before
Binding order matters. The IBO must be bound while the VAO is active. The VAO records which element buffer is attached. If you call glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, 0) while the VAO is still bound, you remove the reference and the draw call will read from the wrong location. Unbind the VAO first, then optionally unbind other buffers.

Also add glDeleteBuffers(1, &ibo) to the cleanup section after the render loop, alongside the existing VBO cleanup.

Two rules for uniforms. First: always retrieve the location after the program has been linked - not before. Second: always call glUseProgram before any glUniform call. If either rule is broken the value is silently ignored and the quad appears black (the default for an unset vec4 uniform is all zeros).
