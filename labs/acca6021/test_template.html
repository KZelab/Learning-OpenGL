Test Template

Overview

This lab gives you a blank, subclass of Tests that you can copy as the starting point for every future rendering experiment.

Rename TestTemplate to something meaningful.

Tasks

Task 1 - Create the Header and Generate the Implementation

  1. In Visual Studio's Solution Explorer, right-click the src/tests folder and choose Add > Class. Name the class TestTemplate. Visual Studio creates both TestTemplate.h and TestTemplate.cpp and registers them in the project automatically.
  2. Open TestTemplate.h and replace its contents with the skeleton below.
  3. With the header open, click on any of the method declarations (for example Update). A lightbulb or screwdriver icon appears in the left margin. Click it and choose Create matching definitions in TestTemplate.cpp. Visual Studio writes all the empty function bodies into the .cpp for you.
  4. Open TestTemplate.cpp to confirm the stubs are there, then build (Ctrl+Shift+B) and confirm it compiles with no errors.
src/tests/TestTemplate.h
#pragma once
#include "Tests.h"

namespace test
{
    class TestTemplate : public Tests
    {
    public:
        TestTemplate();
        ~TestTemplate();

        // TODO: Override only the methods your scene needs.
        // All three are shown here so nothing is forgotten.
        void Update(float deltaTime) override;
        void Render() override;
        void RenderGUI() override;

    private:
        // TODO: Add per-scene member variables here.
        // Examples: Shader, VertexArray, Texture, glm::mat4 ...
    };
}

Task 2 - Register the Test in Main.cpp

  1. Open src/Main.cpp.
  2. Near the top of the file, alongside the other test includes, add: 
    #include "tests/TestTemplate.h"
  3. Scroll down to the block of RegisterTest calls. Add your test alongside the others: 
    TestMenu->RegisterTest<test::TestTemplate>("My Test");
