Lab 17 - RayCasting

Overview

This lab builds an interactive mouse-picking system that works with both spheres and axis-aligned cubes. You will cast a ray from the camera through the cursor into 3D space, dispatch each scene object to the appropriate intersection test, and highlight the nearest hit object in green. Spheres use the quadratic discriminant test from the original version of this lab. Cubes add a second algorithm - the slab method - which treats the box as the overlap of three axis-aligned slabs and finds where the ray enters and exits each one. The lab builds on the projection matrices from the Projections lab (Lab 08) and the test harness registration pattern from the Test Harness lab (Lab 15).

Learning Objectives

By the end of this lab you will be able to:

  • Convert a 2D mouse screen position to a 3D normalised ray in world space using inverse projection and view matrices
  • Apply the quadratic discriminant test to detect a ray-sphere intersection
  • Apply the slab method to detect a ray-AABB intersection for axis-aligned cubes
  • Dispatch each scene object to the correct intersection test based on its ObjectType
  • Maintain separate VAO and IBO pairs for sphere and cube geometry and draw them selectively each frame

Concept

Without a reverse projection, there is no way to determine which 3D object the mouse cursor is hovering over. The GPU rasteriser maps geometry from world space onto the screen during rendering, but nothing in the standard OpenGL pipeline inverts that mapping automatically. Ray casting performs the inversion by hand. A ray is fired from the camera position through the pixel under the cursor, and each scene object is tested to see whether the ray hits it.

The ray originates as a 2D mouse position and must be transformed into world space through three coordinate systems. The mouse position is first mapped from window pixels to Normalised Device Coordinates (NDC), where the screen centre is (0, 0) and the corners sit at -1 and 1. The horizontal conversion is (2.0f * mouseX) / width - 1.0f. The vertical conversion flips Y because window coordinates grow downward while NDC grows upward: 1.0f - (2.0f * mouseY) / height. This NDC position is packed into a glm::vec4 with z set to -1 (pointing forward into the scene) and w set to 1, forming the clip-space ray. Multiplying by glm::inverse(projectionMatrix) converts to eye space. Overriding w to 0 marks the result as a direction, not a point. A final multiplication by glm::inverse(viewMatrix) gives the ray direction in world space.

With the world-space ray, each object is tested with the algorithm that matches its geometry. Spheres use the ray-sphere quadratic test. Writing the ray as P = origin + t * direction and substituting into the sphere equation produces a quadratic in t with coefficients a = dot(d, d), b = 2 * dot(oc, d), and c = dot(oc, oc) - r * r, where oc is the vector from the sphere centre to the ray origin. The discriminant b * b - 4 * a * c gives the answer without solving for t: positive means two intersection points, negative means a miss. Axis-aligned boxes use the slab method. A box is the intersection of three pairs of parallel planes (slabs), one pair per axis. For each axis, two t values give where the ray crosses the near and far plane of that slab. tEnter is the largest of the three near crossings and tExit is the smallest far crossing. When tExit > tEnter and tExit > 0, the ray passes through all three slabs simultaneously and the box is hit. The GLM calls used are glm::inverse, glm::dot, glm::normalize, glm::min, and glm::max.

// GenerateSphereData - UV sphere using spherical coordinates (6 floats per vertex).
void testRayCasting::GenerateSphereData(
    std::vector<float>& vertices,
    std::vector<unsigned int>& indices,
    float radius,
    unsigned int longitudeDiv,
    unsigned int latitudeDiv)
{
    for (unsigned int lat = 0; lat <= latitudeDiv; ++lat) {
        float theta    = lat * glm::pi<float>() / latitudeDiv;
        float sinTheta = sin(theta);
        float cosTheta = cos(theta);

        for (unsigned int lon = 0; lon <= longitudeDiv; ++lon) {
            float phi    = lon * 2.0f * glm::pi<float>() / longitudeDiv;
            float sinPhi = sin(phi);
            float cosPhi = cos(phi);

            float x = cosPhi * sinTheta;
            float y = cosTheta;
            float z = sinPhi * sinTheta;

            vertices.push_back(radius * x); vertices.push_back(radius * y); vertices.push_back(radius * z);
            vertices.push_back(x);          vertices.push_back(y);          vertices.push_back(z);
        }
    }

    for (unsigned int lat = 0; lat < latitudeDiv; ++lat) {
        for (unsigned int lon = 0; lon < longitudeDiv; ++lon) {
            unsigned int first  = (lat * (longitudeDiv + 1)) + lon;
            unsigned int second = first + longitudeDiv + 1;
            indices.push_back(first);   indices.push_back(second);   indices.push_back(first + 1);
            indices.push_back(second);  indices.push_back(second + 1); indices.push_back(first + 1);
        }
    }
}
