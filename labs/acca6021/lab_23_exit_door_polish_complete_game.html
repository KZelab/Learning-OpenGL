Lab 23 - Exit Door, Polish, and Complete Game

Overview

This is the final maze lab. You will complete the game by adding a key pickup, an exit door that can only be opened when all coins have been collected and the key has been found, and a second larger level. The door interaction uses a ray-AABB intersection test (the slab method) to check whether the player's crosshair is pointing at the door - a technique you first saw in the Ray Casting lab (Lab 17). The state machine gains a fourth state (LevelComplete) and the HUD is extended to show level progress, key status, and a seed-based maze regeneration button. By the end of this lab TestMaze is a complete two-level first-person game.

Learning Objectives

By the end of this lab you will be able to:

  • Extend the state machine to four states (Playing, LevelComplete, Won, Lost) and route the Lost state to retry the same seed.
  • Implement the slab-method ray-AABB intersection to detect whether the crosshair is aimed at the exit door.
  • Add a key pickup and gate the door-unlock interaction on possessing the key and collecting all coins.
  • Render the locked door as a textured cube and the key as a cyan elongated spinning object.
  • Complete the HUD: level counter, key status, seed display, Regenerate button, and context-sensitive door prompts.

Concept

The exit door needs an interaction that only fires when the player is looking directly at it. The simplest approach is to treat the camera's front vector as a ray and test it against the door's AABB. The slab method does this efficiently by treating the box as the intersection of three pairs of parallel planes (one pair per axis). For each axis the ray enters one plane at parametric distance t1 and exits the other at t2. The ray hits the box only if all three intervals [t1, t2] overlap. The algorithm maintains a running [tMin, tMax] interval; each axis narrows it. If tMin ever exceeds tMax the intervals have diverged and there is no intersection.

Because the crosshair sits at the exact centre of the screen and the camera's front vector getFront() points to that centre, no screen-to-world unprojection is needed. Setting tMax to InteractDistance at the start means the ray is clamped to a short reach, so the player must stand close to the door. Rays parallel to a slab axis (direction component near zero) are handled separately: if the origin is outside the slab the ray can never cross it, so the function returns false immediately.

The multi-level structure is driven by a level index parameter added to LoadLevel. A static DefaultSeed method maps each level index to a fixed seed, giving reproducible mazes on every run. Storing the seed that was used to build the current maze in m_Seed lets the Lost state retry the exact same layout. The Regenerate button replaces m_Seed with a new std::random_device value, which builds a fresh random maze without changing the level.

Task 3 - Implement RayHitsDoor (slab method)

  1. Add RayHitsDoor to TestMaze.cpp using the skeleton below. The ray origin is the camera position and the direction is m_Camera->getFront() - the camera front vector IS the crosshair direction, so no unprojection is needed.
  2. Build the door AABB. The door tile sits at floor level, but we want a full-height box, so add half the wall height to the Y component of the centre, then extend by half the wall height in each Y direction: 
    glm::vec3 doorCenter = m_Map.DoorPosition()
                       + glm::vec3(0.0f, WallHeight * 0.5f, 0.0f);
glm::vec3 boxMin = doorCenter - glm::vec3(half, WallHeight * 0.5f, half);
glm::vec3 boxMax = doorCenter + glm::vec3(half, WallHeight * 0.5f, half);
  3. Initialise tMin = 0 (reject hits behind the camera) and tMax = maxDistance (clamp to interaction reach).
  4. In the per-axis loop:
    1. If the direction component is near zero the ray is parallel to the slab. Return false if the origin is outside the slab on that axis.
    2. Otherwise compute t1 and t2 (entry and exit along this axis), swap them if t1 > t2, then narrow the interval: 
      tMin = std::max(tMin, t1);
tMax = std::min(tMax, t2);
if (tMin > tMax) return false;
  5. Return true after the loop - all three slabs overlapped.
