Lab 20 - Maze Layout and First-Person View

Overview

In this lab you build the foundation of a first-person maze game. You will construct a single unit cube mesh and use it to render an entire 3D world by iterating a 2D character grid, positioning the cube differently for each tile. A grounded first-person Camera gives you WASD movement and mouse look from the moment the test loads. By the end of the lab you will be able to walk through textured corridors and look around freely. Wall collision is added in the Maze Collision lab (Lab 21), and the full game - timer, collectibles, and exit door - follows in Labs 22 and 23.

Learning Objectives

By the end of this lab you will be able to:

  • Build a 24-vertex unit cube mesh with per-face flat normals and 0-to-1 UV coordinates using VertexArray, VertexBuffer, and IndexBuffer.
  • Render a 3D world by iterating a 2D character grid and positioning a shared cube with translate and scale model matrices.
  • Configure a grounded first-person Camera with WASD movement and mouse look using setGrounded().
  • Apply the Model-View-Projection pipeline and Phong lighting uniforms to a scene that draws many objects from one VAO.
  • Register the new TestMaze class and verify it appears in the test selection menu.

Concept

Without a reuse strategy, a maze with hundreds of wall tiles would need hundreds of separate VAOs - one mesh per tile. That wastes GPU memory and makes the rendering loop complicated. The solution is to build one unit cube and draw it many times, supplying a different model matrix each time. OpenGL draws the same triangles but transforms them to a new position and scale because the vertex shader reads u_Model per draw call. This pattern - one shared mesh, many model matrices - is sometimes called instancing, though here we use a simple per-object uniform rather than a GPU instanced draw.

The cube uses 24 vertices rather than the minimum 8. Each face needs its own four vertices so each can carry a different flat outward normal and a full 0-to-1 UV range. A corner shared between three faces would need three different normals at the same position, which a single vertex cannot express. The vertex data is laid out as position (3 floats), normal (3 floats), and texture coordinate (2 floats), matching the layout expected by Maze.shader. Indices (36 in total: 6 faces times 2 triangles times 3 vertices) are generated with a loop rather than typed by hand.

The MazeMap class owns the tile grid. It exposes At(col, row) to read a character, and TileCenter(col, row) to get the world-space XZ centre of that tile. Tile '#' is a wall and gets a full-height block. Every other character is open floor and gets a thin slab. The Camera is put into grounded mode via setGrounded(true, 1.6f), which locks the Y position to eye height and projects WASD movement onto the horizontal plane.

Task 2 - Build the unit cube mesh

  1. Create src/tests/maze/TestMaze.cpp. At the top, copy the include block from the skeleton below - you will add more functions to this file in later tasks.
  2. Implement BuildCubeMesh(). Declare a float vertices[192] array (24 vertices times 8 floats each). For each of the 6 faces, write 4 rows of 8 floats: x, y, z, nx, ny, nz, u, v.
    • Positions span -0.5 to +0.5 on each axis, centred on the origin.
    • Each face's normal points directly outward (e.g. front face: 0, 0, 1).
    • UV runs from (0,0) at the bottom-left corner to (1,1) at the top-right.
    • Winding is counter-clockwise viewed from outside the cube.
  3. Generate 36 indices programmatically. Loop from face = 0 to 5. For each face, compute base = face * 4 and write two triangles into indices[face * 6 .. face * 6 + 5]:
    • Triangle 1: base+0, base+1, base+2
    • Triangle 2: base+2, base+3, base+0
  4. Create a VertexArray and VertexBuffer. Build a VertexBufferLayout, push float(3) (position), float(3) (normal), and float(2) (texCoord). Call AddBuffer then create the IndexBuffer with 36 indices.

Task 4 - Render the tile grid

  1. Add DrawCube and Render to TestMaze.cpp using the skeleton below.
  2. In DrawCube, bind the shader and set u_Model and u_ObjectColour uniforms. If a texture is supplied, bind it to slot 0, set u_Texture = 0 and u_UseTexture = 1. Otherwise set u_UseTexture = 0. Call m_Renderer.Draw at the end.
  3. In Render, clear the framebuffer, compute the framebuffer aspect ratio, and build the perspective projection matrix with glm::perspective (FOV from m_Camera->getFOV(), near 0.1, far 100). Retrieve the view matrix from the camera.
  4. Bind the shader once and set all the frame-constant uniforms: u_View, u_Projection, u_Light.Position (above the maze centre at y = 8), u_Light.Colour (white), u_CameraPosition, and the four Phong scalars (ambient 0.35, diffuse 0.8, specular 0.2, shininess 32).
  5. Write the tile loop: iterate every (row, col) pair. Get the world-space centre with m_Map.TileCenter(col, row) and the tile character with m_Map.At(col, row). For '#', translate up by half the wall height and scale to (tile, WallHeight, tile). For everything else, translate down slightly and scale to (tile, 0.1f, tile) for a floor slab.
  6. Register the test in src/Main.cpp:
