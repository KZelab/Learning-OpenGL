Lab 18 - Texture2D

Overview

This lab brings together every abstraction class built in the previous series - VertexArray, VertexBuffer, IndexBuffer, Shader, Texture, and Renderer - into a complete test scene. The result is a PNG-textured quad drawn twice on screen at positions controlled in real time via Dear ImGui sliders. Along the way you will learn how to store UV texture coordinates alongside position data in a single vertex buffer, how to build a 2D orthographic projection matrix, and how to draw the same geometry at two different positions in a single frame by uploading a distinct model matrix before each draw call.

Learning Objectives

By the end of this lab you will be able to:

  • Store 2D position and UV texture coordinate data together as two attributes in one interleaved vertex buffer
  • Build an orthographic projection matrix with glm::ortho and multiply it with a per-object model matrix to form a complete MVP uniform
  • Load a PNG image with the Texture class and link it to a sampler2D uniform in GLSL
  • Draw the same quad geometry twice in one frame by computing a fresh model matrix from each translation vector before each draw call
  • Expose real-time position controls using Dear ImGui SliderFloat3 widgets

Concept

Without a transformation matrix, every vertex drawn by the GPU stays at its declared local coordinates. Placing the same quad at two different positions on screen is impossible unless the CPU tells the GPU a unique offset for each copy. A Model-View-Projection (MVP) matrix solves this problem. It is a single 4x4 matrix that carries a vertex all the way from local object space to clip space in one multiplication, and it is uploaded to the shader as a uniform mat4 whose value can be changed between draw calls.

The MVP is the product of three components. The model matrix places the object in the world. In this lab it is a translation built with glm::translate. The view matrix moves the scene to simulate a camera. For this 2D scene the view is an identity matrix, meaning the camera sits at the origin facing the XY plane. The projection matrix maps the 3D scene onto the 2D screen. glm::ortho builds an orthographic projection that maps a chosen rectangle of world coordinates directly to clip space with no perspective distortion. Passing pixel-sized bounds (0 to 960 in width, 0 to 540 in height) means object coordinates correspond to pixel-like screen positions, which makes layout straightforward.

UV coordinates are a second per-vertex attribute stored in the same vertex buffer immediately after the position. Each UV pair is a 2D value from 0 to 1 that pins the vertex to a corner of the texture image. The VertexBufferLayout class records both attributes with two calls to Push<float>(2). The GPU uses this layout to unpack the interleaved data automatically. Inside the fragment shader, a sampler2D uniform receives the texture unit index set from C++ with setUniform1i. The built-in texture() function then returns the texel colour at the interpolated UV coordinate for each fragment.
