ACCA6021 - Graphics and Games Programming

Lab 06 - Textures

Overview

The Index Buffers and Uniforms lab (Lab 05) gave the quad a runtime colour through a uniform vec4. The colour was still a flat solid. This lab replaces that flat colour with a sampled image by introducing the texture pipeline: UV coordinates stored per vertex, an image loaded from disk with stb_image, an OpenGL texture object on the GPU, and a sampler2D uniform that lets the fragment shader look up the right pixel for each fragment.

By the end of the lab your quad will display a loaded PNG with correct orientation and alpha blending.

Learning Objectives

By the end of this lab you will be able to:

  • Extend vertex data with UV coordinates and configure a second vertex attribute pointer for them.
  • Write a GLSL vertex shader that passes UV coordinates to the fragment stage as a varying.
  • Load an image from disk using stb_image and upload it to the GPU with glTexImage2D.
  • Configure texture filtering and wrapping parameters with glTexParameteri.
  • Sample a texture in a fragment shader using a sampler2D uniform and the texture() function.
  • Enable alpha blending so transparent pixels in the image render correctly.

Concept

So far every fragment has been coloured by a value computed entirely in the shader. A texture is a 2D image stored on the GPU that the fragment shader can query to retrieve colour data. Without textures, representing surface detail such as wood grain, skin, or printed labels would require thousands of extra vertices. With textures, a single quad can display an arbitrarily detailed image while remaining just four vertices in the VBO.

The connection between vertices and image pixels is made through UV coordinates (also called texture coordinates). Each vertex carries a vec2 UV value in addition to its position. UV coordinates range from 0.0 to 1.0 across each axis regardless of the image resolution: (0,0) is the bottom-left corner and (1,1) is the top-right. The GPU interpolates the UV across each triangle so that every fragment receives a unique coordinate, which the shader uses to look up the correct texel. This lookup is performed with the built-in GLSL function texture(sampler, uv), where sampler is a sampler2D uniform bound to a texture unit.

On the C++ side, stb_image is a single-header library that decodes common image formats (PNG, JPG, BMP) into a raw byte array. You call stbi_set_flip_vertically_on_load(1) before loading because OpenGL expects the first row of pixel data to be the bottom of the image, while most image files store rows top-to-bottom. The decoded bytes are uploaded to the GPU with glTexImage2D. You then configure how OpenGL samples the texture when it is displayed at a different size with glTexParameteri: the GL_TEXTURE_MIN_FILTER and GL_TEXTURE_MAG_FILTER parameters control interpolation, and the GL_TEXTURE_WRAP_S and GL_TEXTURE_WRAP_T parameters control what happens when UV coordinates fall outside the 0-1 range. Once the texture is on the GPU, the CPU-side image data can be freed with stbi_image_free.

  1. Replace the vertices array in src/main.cpp with the 4-float-per-vertex layout below. Each row contains x, y, u, v.
  2. Update the glBufferData call so the size argument still uses sizeof(vertices) rather than a hard-coded number.

Task 3 - Load a Texture with stb_image

Before the render loop, load the image from disk and upload it to the GPU as an OpenGL texture object. Place any PNG or JPG at res/Textures/1.png.

  1. Add #include "vendor/stb_image.h" near the top of src/main.cpp. The implementation is already compiled in src/vendor/stb_image.cpp, so you do not need the #define STB_IMAGE_IMPLEMENTATION guard here.
  2. After the VAO and shader setup, call stbi_set_flip_vertically_on_load(1). This corrects the vertical flip because OpenGL reads rows bottom-up while PNG files store rows top-down.
  3. Call stbi_load with the path and 4 as the channel count to force RGBA output regardless of the source format.
  4. Generate and configure an OpenGL texture using the skeleton below. GL_CLAMP_TO_EDGE prevents a thin border artifact at the quad edges. GL_LINEAR blends neighbouring texels for a smooth result when the quad is scaled.
  5. Free the CPU-side image data: if (imgData) stbi_image_free(imgData);
  6. Add glDeleteTextures(1, &texture) to the cleanup section after the render loop.
src/main.cpp
stbi_set_flip_vertically_on_load(1);
int imgWidth, imgHeight, imgChannels;
unsigned char* imgData = stbi_load("res/Textures/1.png", &imgWidth, &imgHeight, &imgChannels, 4);

unsigned int texture;
glGenTextures(1, &texture);
glBindTexture(GL_TEXTURE_2D, texture);

glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);

glBindTexture(GL_TEXTURE_2D, 0);

Task 4 - Bind the Texture and Set the Sampler Uniform

OpenGL uses texture units to let a single shader sample multiple textures simultaneously. You activate a unit with glActiveTexture(GL_TEXTURE0), bind your texture to it, and tell the shader which unit to read from by setting the sampler2D uniform to the unit index (0 for GL_TEXTURE0).

  1. After the shader program is created but before the render loop, enable alpha blending so transparent pixels render correctly: 
    glEnable(GL_BLEND);
glBlendFunc(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA);
  2. Still before the loop, link the sampler uniform to texture unit 0. The uniform must be set while the program is active: 
    glUseProgram(shader);
int texLoc = glGetUniformLocation(shader, "u_Texture");
glUniform1i(texLoc, 0);
glUseProgram(0);
  3. Inside the render loop, before the draw call, activate unit 0 and bind your texture: 
    glActiveTexture(GL_TEXTURE0);
glBindTexture(GL_TEXTURE_2D, texture);
  4. Build and run. Your image should appear on the quad. If it is upside-down, confirm that stbi_set_flip_vertically_on_load(1) is called before stbi_load.
