Lab 22 - Collectibles, Timer, and Win Condition

Overview

In this lab you add a game loop to the maze from the previous two labs. The player now has a countdown timer and must collect every gold coin before time runs out. The test class gains a three-state machine: Playing, Won, and Lost. Coins are rendered as small spinning, bobbing cubes and disappear when walked over. A HUD panel shows the remaining time and coin count, and full-screen banners appear when the game ends. The key and exit door mechanics that complete the game are added in the final maze lab (Lab 23).

Learning Objectives

By the end of this lab you will be able to:

  • Implement a three-state game state machine (Playing, Won, Lost) driven by a countdown timer and a win condition.
  • Detect walk-over collectibles using an XZ-only distance check, ignoring the Y axis because pickups bob up and down.
  • Apply time-based animation (sin for bobbing, angle accumulation for spinning) to render coins as dynamic objects.
  • Use edge detection on the Enter key so the restart action fires once per press rather than every frame.
  • Render HUD overlays using ImGui's foreground draw list to draw a crosshair and centred win/lose banners on top of the 3D viewport.

Concept

A game loop is a state machine: the game is always in exactly one state, and each state determines what logic runs each frame and what transitions are possible. Here the states are Playing (normal gameplay), Won (all coins collected), and Lost (timer expired). The state is stored as an enum class member and a switch in Update() branches to the appropriate logic.

Coin pickup uses a simple proximity test. Each Collectible has a world-space position (set by MazeMap::Load when it parses the 'C' tiles). Every frame we compare the player's XZ position to each uncollected coin's XZ position. Y is deliberately excluded because coins bob vertically: the player would miss them if the height had to match exactly. The distance formula is sqrt(dx*dx + dz*dz), and if it falls below a small pickup radius the coin is marked collected.

Edge detection prevents a key press from firing many times while the key is held. The pattern is to store whether the key was pressed last frame: bool advancePressed = advance && !m_AdvancePressedLastFrame. This makes advancePressed true only on the single frame where the key transitions from released to pressed. The same technique is used for any "act on press" interaction, such as the E-key door unlock in the next lab. Coin animation is driven by glfwGetTime(): a sine wave controls the vertical bob and a linearly growing angle drives the spin, both offset by a per-coin phase so each coin moves independently.
