Lab 16 - ClearColour Test and ImGui

Overview

This lab implements testClearColour - the first concrete test scene in the project. Building on the Test Template lab (Lab 16), you will override all three virtual methods, add a Dear ImGui colour picker widget, and register the test in the application menu. When complete, selecting the test from the menu will display an interactive colour picker that changes the window background colour in real time with no recompile needed.

Learning Objectives

By the end of this lab you will be able to:

  • Use ImGui::ColorEdit4 to bind a float[4] array to an interactive colour picker widget
  • Override Update, Render, and RenderGUI in a concrete Tests subclass
  • Register a new test scene with TestMenu::RegisterTest in Main.cpp
  • Trace the data flow from an ImGui widget interaction through to a live glClearColor call

Concept

Without a way to change parameters at runtime, every tweak to a colour, size, or position requires a recompile. For a complex renderer this quickly becomes slow and disruptive. Dear ImGui solves this by providing an immediate-mode GUI framework that lets you expose any C++ variable as an interactive widget, rebuilt every frame.

In the immediate-mode model, there are no persistent widget objects. Instead, RenderGUI() is called every frame and you call ImGui functions directly inside it. ImGui reads the current value of the bound variable, draws the widget, and if the user interacts with it, writes the new value back before the function returns. The application then reads that updated variable on the very next Render() call - no callbacks, no events, no signal/slot wiring.

ImGui::ColorEdit4 takes a label string and a pointer to a float[4] array holding red, green, blue, and alpha in the range 0.0 to 1.0. In Render(), glClearColor reads from that same array and glClear(GL_COLOR_BUFFER_BIT) applies the colour. The pattern - expose a member variable to ImGui in RenderGUI(), read it in Render() - is how every future test scene in this project will work.

Task 3 - Register the test in Main.cpp

  1. Open src/Main.cpp and find the block where other tests are registered with TestMenu->RegisterTest.
  2. Add this include near the top of the file, alongside the other test includes: 
    #include "tests/testClearColour.h"
  3. Add this line in the test registration block: 
    TestMenu->RegisterTest<test::testClearColour>("Test Clear Colour");
  4. Build and run. Select Test Clear Colour from the menu. You should see an ImGui colour picker widget - drag the sliders or click the colour swatch and watch the background update live.
