Lab 13 - Texture Class

Overview

The Shader Class lab (Lab 12) removed the last raw shader management code from main.cpp. Texture loading is the final piece of boilerplate to extract. In earlier labs, loading an image required calling stbi_load, creating a texture object with glGenTextures, setting four parameters, uploading pixel data with glTexImage2D, and remembering to call stbi_image_free and glDeleteTextures at the right time. This lab wraps all of that in a Texture class that follows the same RAII pattern as the buffer classes. After this lab, the entire resource setup that fills main.cpp can be replaced with a handful of class instantiations.

Concept

Loading an image with stbi_load gives you a pointer to raw pixel bytes in CPU memory. OpenGL cannot access CPU memory during rendering - it needs a copy on the GPU. The glGenTextures function creates a texture object (identified by an unsigned int handle, like all GL objects), and glTexImage2D copies the pixel data from CPU to GPU. Once the upload is complete, the CPU-side buffer is no longer needed and stbi_image_free releases it. OpenGL reads texture rows from bottom to top (UV origin at the bottom-left corner), but stbi_load delivers rows from top to bottom. Calling stbi_set_flip_vertically_on_load(1) before loading corrects this mismatch so textures appear the right way up.

glTexImage2D takes two distinct format arguments that are often confused. The internal format (GL_RGBA8) describes how the GPU should store the texture - four channels, eight bits each. The source format (GL_RGBA plus GL_UNSIGNED_BYTE) describes the layout of the CPU data you are uploading. You can choose them independently: for example, you could upload 8-bit-per-channel data and ask the GPU to store it at 16-bit precision. The four glTexParameteri calls set the sampling behaviour: GL_TEXTURE_MIN_FILTER controls how the GPU samples when the texture is displayed at smaller than native resolution (with filtering to avoid aliasing), GL_TEXTURE_MAG_FILTER controls the same for larger-than-native display, and GL_TEXTURE_WRAP_S/T control what happens when UV coordinates go outside the 0-1 range. GL_CLAMP_TO_EDGE clamps the coordinate to the nearest edge pixel.

OpenGL supports up to 80 texture units (slots) that can be bound simultaneously, numbered GL_TEXTURE0 through GL_TEXTURE79. Before binding a texture for use in a draw call, you must first activate the target unit with glActiveTexture(GL_TEXTURE0 + slot), then bind the texture with glBindTexture. The corresponding GLSL sampler2D uniform must be set to the same integer slot via shader.setUniform1i("u_Texture", 0). This indirection lets a single draw call read from multiple textures at once - normal maps, diffuse maps, and shadow maps can all be bound to different slots simultaneously. The Texture::Bind(unsigned int slot) method handles both the glActiveTexture and glBindTexture calls together so the caller only specifies the slot number.

Task 2 - Implement the Texture Upload and Bind

  2. In the constructor, call stbi_set_flip_vertically_on_load(1) before stbi_load. The fourth argument to stbi_load is the number of desired channels - pass 4 to force RGBA output regardless of the source image format.

  4. Call glTexImage2D with GL_RGBA8 as the internal format and GL_RGBA, GL_UNSIGNED_BYTE as the source format. Use the width and height values returned by stbi_load.
  5. Unbind the texture after upload, then free the CPU buffer with stbi_image_free(m_Localbuffer).
  6. In Bind(slot), call glActiveTexture(GL_TEXTURE0 + slot) first, then glBindTexture.
