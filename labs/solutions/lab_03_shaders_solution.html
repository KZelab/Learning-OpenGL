Lab 03 - Shaders [SOLUTION]

Overview

In the First Triangle lab (Lab 02) you relied on OpenGL's built-in fixed-function pipeline to colour the triangle white. This lab replaces that with your own GLSL shader program. You will write a vertex shader and a fragment shader, store them together in a single .shader file, write a C++ parser to load them, compile both stages with error checking, and link them into a program that drives every pixel of the triangle.

The project also returns to Core Profile, which disables the fixed-function fallback entirely. From Lab 03 onward, all rendering requires an explicit shader program.

Prerequisites

  • Lab 02 complete: a white triangle renders in the centre of the window using a VAO and VBO.
  • The VAO and VBO setup code from Lab 02 is present in src/main.cpp and working.
  • The res/shaders/ directory exists in the project root (create it if not present).

Learning Objectives

By the end of this lab you will be able to:

  • Describe the roles of the vertex shader and fragment shader within the OpenGL pipeline.
  • Write basic GLSL for both stages and explain the purpose of gl_Position and a fragment output variable.
  • Parse a custom .shader file format to separate vertex and fragment source at runtime.
  • Compile shader stages and read driver error messages using glGetShaderInfoLog.
  • Link a shader program and activate it before a draw call with glUseProgram.

Concept

Switching back to Core Profile removes the fixed-function pipeline entirely. Every draw call now requires an active shader program or nothing renders. A program contains at minimum two stages. The vertex shader runs once for each vertex the draw call submits. It receives the position from attribute slot 0 and must write a clip-space position to the built-in output gl_Position. OpenGL then determines which screen pixels the assembled triangle covers.

The fragment shader runs once for each of those covered pixels. It must write a colour to a user-declared out vec4 variable. Both shaders are written in GLSL and compiled at runtime by the GPU driver via glCreateShader, glShaderSource, and glCompileShader. Compile errors are returned as strings via glGetShaderInfoLog. The two compiled shaders are attached to a program object with glAttachShader and linked with glLinkProgram. Calling glUseProgram before a draw call activates that program for the pipeline.

Rather than managing two separate source files per shader, this project uses a single .shader file with custom section markers. The lines #shader vertex and #shader fragment are not valid GLSL. They are directives read by a C++ parser that distributes each subsequent line to the correct source string. The OpenGL compile API receives two clean GLSL strings with no trace of the markers.

The fragment shader outputs a flat blue colour defined by the vec4 constructor: red 0.2, green 0.6, blue 1.0, alpha 1.0. Change these values to any colour you prefer. 

#shader vertex
#version 430 core
layout (location = 0) in vec3 aPos;
void main() {
    gl_Position = vec4(aPos, 1.0);
}

#shader fragment
#version 430 core
out vec4 FragColour;
void main() {
    FragColour = vec4(0.2, 0.6, 1.0, 1.0);
}
GLSL quick reference. #version 430 core selects GLSL 4.30 with Core Profile semantics. layout (location = 0) in vec3 aPos declares an input attribute at slot 0, matching the index you passed to glVertexAttribPointer in Lab 02. gl_Position is a built-in vec4 output of the vertex shader that holds the final clip-space position. out vec4 FragColour declares the fragment shader's colour output. vec4(r, g, b, a) constructs a four-component float vector.

Task 4 - Add the Shader Compiler with Error Checking

Copy the function below into src/main.cpp, above main() and after the parser. It compiles one shader stage and prints any GLSL error message the driver reports. The function is provided in full because the error-checking API is boilerplate you will reuse unchanged in every project. 

// file: src/main.cpp -- provided for reference, add above main()
static unsigned int compileShader(unsigned int type, const std::string& source) {
    unsigned int id = glCreateShader(type);
    const char* src = source.c_str();
    glShaderSource(id, 1, &src, nullptr);
    glCompileShader(id);

    int result;
    glGetShaderiv(id, GL_COMPILE_STATUS, &result);
    if (!result) {
        int length;
        glGetShaderiv(id, GL_INFO_LOG_LENGTH, &length);
        char* message = new char[length];
        glGetShaderInfoLog(id, length, &length, message);
        std::cerr << "Shader compile error: " << message << '\n';
        delete[] message;
        return 0;
    }
    return id;
}

To test the error reporting, try introducing a deliberate typo in the fragment shader (for example, remove the semicolon after FragColour = ...), run the project, and read the console output. Fix the typo before continuing.

src/main.cpp

// file: src/main.cpp -- complete solution for Lab 03

#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <fstream>
#include <sstream>
#include <iostream>
#include <string>

// holds the two GLSL source strings extracted from a .shader file
struct ShaderSource {
    std::string vertex;
    std::string fragment;
};

// reads a .shader file and splits it at #shader vertex / #shader fragment markers
ShaderSource parseShader(const std::string& filepath) {
    std::ifstream file(filepath);
    std::stringstream ss[2];
    int type = -1;
    std::string line;
    while (std::getline(file, line)) {
        if (line.find("#shader vertex") != std::string::npos)
            type = 0;
        else if (line.find("#shader fragment") != std::string::npos)
            type = 1;
        else if (type != -1)
            ss[type] << line << '\n';
    }
    return { ss[0].str(), ss[1].str() };
}

// compiles one shader stage and prints any driver error to stderr
static unsigned int compileShader(unsigned int type, const std::string& source) {
    unsigned int id = glCreateShader(type);
    const char* src = source.c_str();
    glShaderSource(id, 1, &src, nullptr);
    glCompileShader(id);

    int result;
    glGetShaderiv(id, GL_COMPILE_STATUS, &result);
    if (!result) {
        int length;
        glGetShaderiv(id, GL_INFO_LOG_LENGTH, &length);
        char* message = new char[length];
        glGetShaderInfoLog(id, length, &length, message);
        std::cerr << "Shader compile error: " << message << '\n';
        delete[] message;
        return 0;
    }
    return id;
}

// compiles both stages, links them into a program, returns the program ID
static unsigned int createProgram(const std::string& vertSrc, const std::string& fragSrc) {
    unsigned int program = glCreateProgram();
    unsigned int vs = compileShader(GL_VERTEX_SHADER, vertSrc);
    unsigned int fs = compileShader(GL_FRAGMENT_SHADER, fragSrc);
    glAttachShader(program, vs);
    glAttachShader(program, fs);
    glLinkProgram(program);
    glDeleteShader(vs);
    glDeleteShader(fs);
    return program;
}

int main()
{
    // initialise GLFW and target OpenGL 4.3 Core Profile
    if (!glfwInit()) { std::cerr << "GLFW init failed\n"; return -1; }
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 4);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);

    GLFWwindow* window = glfwCreateWindow(800, 600, "OpenGL Window", nullptr, nullptr);
    if (!window) { glfwTerminate(); return -1; }
    glfwMakeContextCurrent(window);

    glewExperimental = GL_TRUE;
    if (glewInit() != GLEW_OK) { glfwTerminate(); return -1; }
    std::cout << "OpenGL: " << glGetString(GL_VERSION) << '\n';
    glfwSwapInterval(1);

    // triangle in NDC
    float vertices[] = {
         0.0f,  0.5f, 0.0f,
        -0.5f, -0.5f, 0.0f,
         0.5f, -0.5f, 0.0f
    };

    // VAO records the vertex layout
    unsigned int vao;
    glGenVertexArrays(1, &vao);
    glBindVertexArray(vao);

    // VBO uploads vertex data to the GPU
    unsigned int vbo;
    glGenBuffers(1, &vbo);
    glBindBuffer(GL_ARRAY_BUFFER, vbo);
    glBufferData(GL_ARRAY_BUFFER, sizeof(vertices), vertices, GL_STATIC_DRAW);

    // attribute 0 = vec3 position, tightly packed
    glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3 * sizeof(float), (void*)0);
    glEnableVertexAttribArray(0);
    glBindVertexArray(0);

    // load and compile the shader from the .shader file
    ShaderSource src = parseShader("res/shaders/triangle.shader");
    unsigned int shader = createProgram(src.vertex, src.fragment);

    // render loop
    while (!glfwWindowShouldClose(window))
    {
        glClearColor(0.1f, 0.1f, 0.15f, 1.0f);
        glClear(GL_COLOR_BUFFER_BIT);

        // activate the shader program then draw the triangle
        glUseProgram(shader);
        glBindVertexArray(vao);
        glDrawArrays(GL_TRIANGLES, 0, 3);

        glfwSwapBuffers(window);
        glfwPollEvents();
    }

    // release GPU resources
    glDeleteVertexArrays(1, &vao);
    glDeleteBuffers(1, &vbo);
    glDeleteProgram(shader);
    glfwDestroyWindow(window);
    glfwTerminate();
    return 0;
}
