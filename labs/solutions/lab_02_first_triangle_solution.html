Overview

In the Window and Context lab (Lab 01) you built a render loop that clears the screen each frame. This lab focuses on putting something on that screen for the first time. You will upload triangle geometry from CPU RAM to the GPU using a Vertex Buffer Object (VBO), record how the GPU should interpret it using a Vertex Array Object (VAO), and issue your first draw call with glDrawArrays.

This lab uses OpenGL's built-in fixed-function pipeline so you can concentrate entirely on the geometry upload process. No custom shaders are written here. The Shaders lab (Lab 03) will replace the hardware defaults with your own GLSL programs. The VAO and VBO pattern you establish in this lab is the foundation that every subsequent lab builds on.

Concept

The GPU is a separate processor with its own dedicated memory. Your triangle coordinates exist only in CPU RAM. Before any rendering can happen, that data must cross the system bus into GPU VRAM. A Vertex Buffer Object (VBO) is an OpenGL-managed block of GPU memory that holds raw bytes. You create one with glGenBuffers, select it as the active buffer with glBindBuffer(GL_ARRAY_BUFFER, vbo), then copy your vertex array into it with glBufferData. The GL_STATIC_DRAW hint tells the driver the data will not change often, so it can place the buffer in fast GPU-local memory.

Raw bytes in a buffer have no inherent meaning. Nine floats could be three RGB colours, three XYZ positions, or something else entirely. A Vertex Array Object (VAO) stores the interpretation rules. While a VAO is bound, each call to glVertexAttribPointer records how one attribute should be read from the currently-bound VBO: how many components it has, what numeric type they are, and how many bytes apart consecutive vertices are (the stride). Calling glEnableVertexAttribArray activates the slot. Binding the VAO again later restores all these rules in a single call.

When the draw call glDrawArrays(GL_TRIANGLES, 0, 3) runs, the GPU reads three vertices from the bound VAO and assembles them into a triangle. For this to produce pixels, OpenGL needs a program to run on each vertex and each covered pixel. Writing that program in GLSL is the subject of Lab 03. For now, this lab uses OpenGL's Compatibility Profile, which includes a built-in fixed-function pipeline. When no custom shader program is bound, the GPU falls back to hardware defaults that colour geometry solid white and apply no transformation. This lets you learn the geometry upload process in isolation, without any shader code.

Note - Hardware Fixed-Function Pipeline. The triangle in this lab renders white because OpenGL's built-in fixed-function pipeline assigns a default white colour when no shader is active. This is intentional. The Shaders lab (Lab 03) will replace this behaviour with a GLSL fragment shader that lets you choose any colour. At that point the project also switches back to the Core Profile, which requires custom shaders and disables the fixed-function defaults entirely.

Task 5 - Describe the Vertex Layout

Tell the GPU how to interpret the bytes in the VBO. Each vertex is three tightly packed floats (x, y, z), so the stride is 3 * sizeof(float) and the offset is zero. In Compatibility Profile, generic attribute slot 0 is aliased to the vertex position used by the fixed-function pipeline, so no shader is needed to make this work.

  1. Call glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3 * sizeof(float), (void*)0).
    • Index 0 - attribute slot (position)
    • Size 3 - three components per vertex (x, y, z)
    • Type GL_FLOAT
    • Normalised GL_FALSE - values are already in NDC, do not rescale
    • Stride 3 * sizeof(float) - 12 bytes between consecutive vertices
    • Offset (void*)0 - position data starts at byte 0
  2. Call glEnableVertexAttribArray(0) to activate attribute slot 0.
  3. Call glBindVertexArray(0) to unbind the VAO now that setup is complete.

Solution

// file: src/main.cpp -- complete solution for Lab 02

#include <GL/glew.h>
#include <GLFW/glfw3.h>
#include <iostream>

int main()
{
    // initialise GLFW
    if (!glfwInit()) { std::cerr << "GLFW init failed\n"; return -1; }
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 4);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    // Compatibility Profile gives access to the fixed-function pipeline (no shader required)
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_COMPAT_PROFILE);

    // create window and bind its OpenGL context to this thread
    GLFWwindow* window = glfwCreateWindow(800, 600, "OpenGL Window", nullptr, nullptr);
    if (!window) { glfwTerminate(); return -1; }
    glfwMakeContextCurrent(window);

    // load OpenGL function pointers via GLEW
    glewExperimental = GL_TRUE;
    if (glewInit() != GLEW_OK) { glfwTerminate(); return -1; }
    std::cout << "OpenGL: " << glGetString(GL_VERSION) << '\n';
    glfwSwapInterval(1);

    // triangle in NDC: x and y range from -1.0 to 1.0, z = 0.0 (on the near plane)
    float vertices[] = {
         0.0f,  0.5f, 0.0f,   // top centre
        -0.5f, -0.5f, 0.0f,   // bottom left
         0.5f, -0.5f, 0.0f    // bottom right
    };

    // create the VAO first so it records everything set up below
    unsigned int vao;
    glGenVertexArrays(1, &vao);
    glBindVertexArray(vao);

    // create a VBO and upload the vertex data to the GPU
    unsigned int vbo;
    glGenBuffers(1, &vbo);
    glBindBuffer(GL_ARRAY_BUFFER, vbo);
    glBufferData(GL_ARRAY_BUFFER, sizeof(vertices), vertices, GL_STATIC_DRAW);

    // describe the layout: attribute 0 holds a vec3 position, tightly packed
    // in Compatibility Profile, attribute 0 maps to the fixed-function vertex position
    glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3 * sizeof(float), (void*)0);
    glEnableVertexAttribArray(0);
    glBindVertexArray(0);

    // render loop
    while (!glfwWindowShouldClose(window))
    {
        glClearColor(0.1f, 0.1f, 0.15f, 1.0f);
        glClear(GL_COLOR_BUFFER_BIT);

        // no glUseProgram: the fixed-function pipeline renders the triangle white
        glBindVertexArray(vao);
        glDrawArrays(GL_TRIANGLES, 0, 3);

        glfwSwapBuffers(window);
        glfwPollEvents();
    }

    // release GPU resources before exit
    glDeleteVertexArrays(1, &vao);
    glDeleteBuffers(1, &vbo);
    glfwDestroyWindow(window);
    glfwTerminate();
    return 0;
}
